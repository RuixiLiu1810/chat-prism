# Citation 论证结构工作流（冻结版 v3）

更新时间：2026-03-30

## 0. 背景修订（来自评审）
本版接受以下关键批评并调整执行顺序：
1. `need` 基础分类器未经标注校准，不能直接作为强门控地基。
2. 段落模式工程量大、ROI 不明，不能抢占主线优先级。
3. “support 与 similarity 分离重排”属于增量收益，不应早于可用性与数据基线。
4. P0 不是算法完备度，而是“Search Citation 的可见反馈与可用闭环”。

## 1. 冻结目标与边界
- 目标：在不牺牲可用性的前提下，把引用推荐从“纯相似检索”升级为“有判定、有证据、有回滚”的稳健流程。
- 非目标（当前不做）：
1. 不做全自动批量插引。
2. 不做全量 LLM 主链路。
3. 不在无数据支撑下启用强门控。

## 2. 最高优先级顺序（硬约束）
1. P0：UX 可用性门禁先通过（加载/结果/错误/重试/关闭反馈完整）。
2. P1：S1 观察层最小落地（仅返回并展示 `need_decision`，不改行为）。
3. P1.5：采集真实用法与标注数据（选区粒度、claim_type 分布、误判样本）。
4. P2：S2 软门控（仅控 auto-apply），且必须基于校准后阈值。
5. P3：S3 类型路由（加权偏置）再评估是否上线。

以下项默认后置（Parked）：
- 段落模式完整 claim 管线（含 claim 级独立展示）。
- S4“support_score 与 similarity_score 深度拆分重排”。

## 3. 分类器风险策略（必须执行）
- 原则：False Negative 成本高于 False Positive。
- 在未校准前：
1. 禁止用 `need=no` 做硬阻断检索。
2. 禁止用未校准 controversy 词表做强决策。
3. 不确定样本默认提升到 `suggest`（宁可多给 review，不可静默漏引）。
- 词表策略：
1. 去除过泛噪声触发词（如单独 `however`）。
2. 争议判定必须依赖复合信号（转折 + 结论冲突/不一致表述），而非单词命中。

## 4. 段落模式策略（降级为受控实验）
- 默认行为：继续以用户选区为主，不假设用户总是整段操作。
- 若输入为大段文本：
1. 先做轻量 claim 切分与去重。
2. 仅处理高优先 claim（预算受限），不做逐句全量检索。
3. 不立即建设复杂 claim 级 UI；先在 debug 可观测。
- 触发升级条件：有真实数据证明“整段触发占比高且误检显著”。

## 5. LLM 接入策略（保守）
- 默认主链路：规则 + 轻量语义 + 多源检索。
- LLM 仅在低置信触发：
1. query 质量低。
2. top 结果分歧大（低分/小 gap）。
3. 支持性不足且用户仍要求继续。
- LLM 参与范围：query rewrite / 支持性复核。
- 禁止：LLM 全量 claim 直通主排序。
- 失败处理：必须无损回退到非 LLM 路径。

## 6. 分阶段执行（重排后）

### S0（P0 UX Gate）
- 目标：Search Citation 全链路可见可控。
- 必须满足：触发后有 loading；完成后有 results/error；支持 retry/close；不出现“点击后无反馈”。
- Gate：GUI checklist 通过且无 P0 回归。

### S1（观察层最小版）
- 目标：`need_decision` 进入 `citation_search` 返回，并在 Debug 可见。
- 行为：不改变检索与插入决策。
- Gate：线上行为零回归。

### S1.5（数据基线）
- 目标：收集校准所需数据。
- 数据项：选区长度分布、句/段比例、claim_type 分布、no/suggest/must 分布、误判样本。
- Gate：达到最小样本量后才允许 S2。

### S2（软门控）
- 目标：仅降低 auto-apply 误插，不降低检索可达性。
- 规则：`need=no` 仅禁止 auto-apply，不阻断检索与 review。
- 前提：阈值由样本校准，不允许经验常量直接上线。
- Gate：auto 误插下降，召回不显著下降。

### S3（可选）
- 目标：类型路由加权偏置（不是硬路由）。
- 上线条件：S2 稳定且数据证明有收益。
- Gate：top1 支持率提升并且延迟/成本可接受。

### S4（Parked）
- 支持性深度拆分重排，延后到 S0-S3 稳定后评估。

## 7. 评测与上线门禁
- 指标：
1. 可用性：触发反馈完整率、失败可恢复率。
2. 检索：top1/top3/top5、MRR、empty_rate。
3. 门控：auto 误插率、漏引代理指标。
4. 成本：每次检索 provider 调用数、LLM 触发率、延迟。
- 发布规则：
1. 任一阶段需两批样本稳定。
2. 未过门禁不得改默认值。
3. 出现 P0 回归，立即回滚到上一稳定阶段。

## 8. 与当前 workflow 的耦合
- `TASK_BOARD`：以 S0 -> S1 -> S1.5 -> S2 顺序推进。
- `NEXT_STEPS`：仅保留当前阶段动作，避免并发扩面。
- `PHASE_STATUS`：Phase 8 标注为“v3 收敛执行中”。

## 9. 执行纪律
1. 先修可用性，再做算法复杂化。
2. 先拿数据，再开门控。
3. 无校准，不上线强策略。
4. 对用户风险不对称场景，宁可保守推荐，不可静默漏引。
