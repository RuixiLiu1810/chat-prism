L1 已确定：请选择输入区形态

同样是 L1，输入区设计会显著影响“命令行交互手感”和可发现性。

I1 · 极简单行输入（最像经典 CLI）

... transcript ...
────────────────────────────────────────
> input...

优点：干净、快。缺点：命令可发现性弱，新用户学习成本高。

I2 · 单行输入 + 轻提示栏（推荐）

... transcript ...
────────────────────────────────────────
> input...
  /help  /commands  /model  /approve

优点：保持 CLI 感，同时提高命令可发现性。复杂度低，适合 MVP。

I3 · 多行输入框 + 提示栏

... transcript ...
────────────────────────────────────────
┌──────────────────────────────────────┐
│ input line 1...                      │
│ input line 2...                      │
└──────────────────────────────────────┘
  /help  /commands  /model  /approve

优点：长提示更友好。缺点：更像编辑器，CLI 纯粹感下降，实现复杂度更高。

建议：若目标是快速对标 Claude Code 体验并控制实现风险，先选 I2。
