agent-cli TUI 对标参考（当前 vs Claude 风格）

目标不是“长得像”，而是建立更稳定的信息层级、对话节奏和会话连续性。

当前（日志感）
[streaming] Connected...
[completed] response completed.
[tool_running] Running run_shell_command...
[result] approval required
[turn:suspended]

问题：事件可读性弱，语义层级不稳定，容易“像重开一轮”。

Claude 风格（对话感）
› /help
└ Help dialog dismissed

› read one file
● I’ll inspect the folder first.
└ Waiting for approval: /approve shell once

优势：输入/回复/系统回执三层固定语法，状态不打断会话叙事。

你希望我们先把哪条主线做到位？

选中后我就按该方向继续出下一轮视觉草案与实现规格。

A

对话语法优先

统一前缀体系：› 用户、● 助手、└ 回执，先解决“可读性与手感”。

B

状态机优先

先修会话生命周期：suspended、审批、继续执行，确保同轮不中断。

C

布局框架优先

先做全屏框架：轻量头部 + 对话区 + 输入区 + 状态行，再逐步替换内容渲染。
