## 详细迁移/补齐建议

1. **工具插件化/动态注册**
   - 设计工具注册表与插件API，支持热加载/卸载。
   - 工具目录结构与元数据标准化，便于自动发现与注册。
   - 用户可通过配置/命令行/插件市场扩展工具。

2. **工具生态扩展**
   - 优先补齐批量操作、计划/任务、团队协作、Notebook、Web、LSP、技能发现等高阶工具。
   - 支持多模态输入/输出（如图片、表格、音频等）。

3. **合约-实现一致性与中间层**
   - 统一参数校验、错误处理、审批流、并发策略的中间层。
   - 自动化合约-实现一致性校验，防止实现偏差。

4. **工具调用链路结构化与审批/挂起/追溯**
   - 工具调用消息结构化，支持打标签、队列化、异步挂起/恢复/撤销/历史追溯。
   - 审批/权限分级细化，审批历史持久化、可追溯。

5. **多 agent 协作与快照**
   - 多 agent 状态同步、快照、回滚、协作API。
   - 支持多 agent 独立记忆空间与隔离。

6. **记忆系统多级/快照/同步/隔离**
   - 记忆系统支持 user/project/local 多级作用域，快照、同步、回滚、导入导出。
   - 记忆与 agent/会话/项目强绑定，提升数据安全与隔离性。

7. **输入状态机/CLI手感提升**
   - 输入状态机独立，支持命令补全、历史检索、撤销、审批队列。
   - CLI 手感对标 reference，提升流畅度。

8. **并发安全与健壮性**
   - 落地并发调度、冲突检测、超时、重试、熔断等机制。

9. **工具消息流/反馈结构化**
   - 工具调用/审批/错误/状态 slot 化，UI 结构化反馈，提升用户可感知性。
